Minterms and Implicants

Minterms are product terms of boolean variable, e.g. ABC. Could be represented by numbers, i.e. 6 means 110 or ABCˉAB\bar{C}ABCˉ.

Implicants are binary numbers used to represent the logic equation terms. In a situation where one of the variables is missing we use a “-” to keep the order of the variables, e.g. 11-0, 01--.

Tabular Method

Recall if we have

F(X,Y)=∑(2,3)F(X,Y) = \sum(2,3) F(X,Y)=∑(2,3)

minterm 2:10minterm 3:11minterm\ 2: 10\\ minterm\ 3: 11 minterm 2:10minterm 3:11

∴F(X,Y)=XYˉ+XY=X(Yˉ+Y)=X\therefore F(X,Y)=X\bar{Y}+XY=X(\bar{Y}+Y)=X ∴F(X,Y)=XYˉ+XY=X(Yˉ+Y)=X

This combines 10 and 11 into 1-. The dash symbolizes the term that we do not care about.

Then we devise the tabular method, in which we keep combining the minterms to form dash-denoted newer terms until exhaustion. Then we took the un-combined terms / remaining terms and sum these.

Example: Simplify F(W,X,Y,Z)=∑(0,2,5,6,7,8,10,14)F(W,X,Y,Z) = \sum(0,2,5,6,7,8,10,14)F(W,X,Y,Z)=∑(0,2,5,6,7,8,10,14)

Use one single table, list out step 1, step 2 ...

The first step tracks all raw binaries. The second step attempts to combine the terms listed in step 1. Each term in the second step has one dash(-). The third step attempts to combine te terms listed in step 2. Each term in the third step has two dashes(-).

Eventually, we get

Priority Encoder

Used to prevent 2 simultaneous input at once. The inputs have different priorities. When simultaneous input happens, only the input with the highest priority will be recorded.

For a 4 to 2 priority encoder:

Y0Y_0Y0​ Y1Y_1Y1​ Y2Y_2Y2​ Y3Y_3Y3​ AAA BBB
1 0 0 0 0 0
X 1 0 0 0 1
X X 1 0 1 0
X X X 1 1 1

Multiplexers and Demultiplexers

The ancient manual multiplexing switch selector a mechanical rotary switch. In digital electronics, multiplexers (MUX) are also known as data selectors because they can select each input line. Generally, the selection of input lines in a multiplexer is controlled by an additional set of inputs called control lines. According to the binary conditionof these control inputs, the appropriate input line is connected directly to the output. The opposite is true for demultiplexers (DeMUX).

Normally, a multiplexer has 2n2^n2n input lines and nnn control lines.
