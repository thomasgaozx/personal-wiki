Binary-Coded-Decimal (BCD) Code

Binary-Coded-Decimal is often used to simplify conversion from decimal numbers to binary.

Negative Numbers and 2s Complement

The digit ahead of the most significant bit is 0 for positive numbers, and 1 for negative numbers.

1's Complement is the complete invert of a positive number, e.g. 1001001⟶01101101001001\longrightarrow 01101101001001⟶0110110.

2's Complement is 1's complement plus one.

Minterms and Implicants

Minterms are product terms of boolean variable, e.g. ABC. Could be represented by numbers, i.e. 6 means 110 or ABCˉAB\bar{C}ABCˉ.

Implicants are binary numbers used to represent the logic equation terms. In a situation where one of the variables is missing we use a “-” to keep the order of the variables, e.g. 11-0, 01--.

Multiplexers and Demultiplexers

The ancient manual multiplexing switch selector a mechanical rotary switch. In digital electronics, multiplexers (MUX) are also known as data selectors because they can select each input line. Generally, the selection of input lines in a multiplexer is controlled by an additional set of inputs called control lines. According to the binary conditionof these control inputs, the appropriate input line is connected directly to the output. The opposite is true for demultiplexers (DeMUX).

PLC

Convential relay-based control is hard-wired to performa specific function. When the system requirements change, the physical relay wiring should be changed. PLC eliminates such hard-wiring. Less wiring means higher reliability. The reliability of solid-state components is higher than that of electrical relay. The risk of shock or short circuits are also negligible for PLC. PLC also has increased flexibility, new programs can be generated without change in wiring.
The arhchitecture of a PLC is basically the same as a general computer. However, a well-designed PLC is less affected by electrical noise.Personal computer perate in non-deterministic fashion adn cannot be relied upon to operate in real-time fashion as PLC.

A PLC processes binary input signals (from sensors) to produce output signals that directly control processes (actuators).

Components of PLC

  • Central Processing Unit (CPU) and Memory. The CPU itself is usually a microcontroller.

  • I/O modules (digital or analog)

  • Interfacing Modules： used to communicate with other PLCs or networks through different protocols, such as Serial/Parallel, Ethernet, Profibus etc.

  • Power Supply: provides DC power to other modules athat plug into the PLC's rack.

  • Programming Device: used to enter the desired program into the memory of the processor. A PC is the most commonly used programming device. An IEC standard has been implemented that calls for 5 standard PLC programming languages.

    • Ladder Diagram (LAD) (most popular one due to similarity to conventional electrical diagram)
    • Function Block Diagram (FBD)
    • Structured Text List (STL)
    • Instruction List (IL)
    • Sequential Function Chart (SFC)

  • Fixed I/O is typical of small PLCs whose processor and I/O are packaged together and I/O terminals are available but cannot be expanded. This type of PLCs has lower cost.

  • Modular I/O is divided into components into which separate modules can be plugged. Various numbers of modules can be selected and integrated into a rack.

PLC Mechanisms

  1. Input Module Wiring: connect the input sensors / buttons into the input ports with different input addressing.
  2. Output Module Wiring: connect the output actuators into the output ports with different output addressing.
  3. Work with input address and output address directly (where ladder logic happens).

PLC Programming

  • Program Files are stored in the processor memory.
    • System Files contains system information
    • Reserved is booked by the processor and is not accessible to the user.
    • Main Ladder Program defines how the controller is to operate.
  • Data files stores input and output status, processor status, the state of various bits, and some numerical values.
    • Output stores the state of the output terminals for the controller.
    • Input stores the status of the input terminals for the controller.
    • Status stores controller operation information.
    • Bit is used for internal relay logic storage.
    • Timer stores the timer accumulated and preset values and status bits.
    • Counter stores the counter accumulated and preset values and status bits.
    • Control stores the lengths, pointer position, and status bit for specific instructions such as shift registers and sequencers.
    • integer stores numerical values or bit information.
    • Network communication is used for network communication if installed.

Rockwell PLC Addressing Convention

The following convention for data addressing will be used:

ZXX.YYZXX.YY ZXX.YY

  • ZZZ indicates the type of module in the PLC rack. III denotes input module, OOO denotes output module, TTT denoes a timer, CCC denotes a counter, AIAIAI denotes analog inputs, AOAOAO denotes analog output, and FFF or MMM for flags (memory elements).
  • XXXXXX is the assigned address of the module in the PLC rack.
  • YYYYYY is the bit address or desired I/O number on the module (bit number).

PLC Operating Cycle

During each operating cycle, the processor reads all the inputs and energizes or deenergizes the outputs according to the user program. In one cycle, there are four execution stages:

  1. Self-test: checking integrity of I/O modules.
  2. Read inputs: the current sttaus of each input condition switch is placed in the input status table.
  3. Solve logic (RUN): CPU evaluates the program logic in a browsing mode.
  4. Set outputs: when the logic-slving stage completes, the associated output will be ed into the output status table and transferred to the output modules.

Signals may be missed if it does not occur at the Read Input stage of hte operating cycle. Pulse Stretch Function (Latch) is used to prolong temporary input signal until it is read. Interrupt Function interrupts the current operating cycle and skip to the next input stage.

Ladder Logic

Ladder Diagram Language is by far the most commonly used PLC language. They are called “ladder” diagrams because they resemble a ladder, with two vertical rails (supply power) and as many “rungs” (horizontal lines) as there are control circuits to represent.

  • Contact: Synonym for switches. Contacts stand between power and Coils, asking questions.
  • Rung: horizontal lines.
  • Relay: synonym for output coil

Switch, Output Coil, and Function Block

  • Normally Open Switch: the switch is normally open (with value 0), signal cannot pass through. When the user close the switch (set the value to 1), the signal can pass through.
  • Normally Closed Switch: the switch is normally closed (with value 0), signal can pass through. When the user open the switch (set value to 1), the signal cannot pass through.

  • Normally Open Coil Output: if there is a signal, the coil will become on (set to value 1).
  • Normally Closed Coil Output: if ther is no signal, the coil is activated (has value 1).

  • Function Block: a function refers to the special operation in the PLC such as Timer, Counter, Arithmetic, Bitwise, Conversion, Comparison, etc.. The end user needs to type the parameters and invoke conditions for each function.

Important Rules:

  • Switches can be reused.
  • All input contacts are placed in the left side of the output, the reverse is not allowable.
  • Output coils or functional blocks must not be directly connected to the left line of ladder diagram. An executable condition must be present. Some functions such as END, SBN, and RET are exception.
  • Output coil address should only be used once. However, the contacts of a coil can be used as inputs many times.
  • If there is > 1 output per rung, outputs should be connected in parallel.

TON, TOF, RTO Function

Parameters:

  • Timer: the address of output switch (e.g. T4.1), the output timer
  • Timer Base: interval for each increment (e.g. 0.01).
  • Preset: the incrementation count limit that "Accumulated" parameter is aiming for, the max time is Timer Base×PresetTimer\ Base\times PresetTimer Base×Preset.
  • Accumulated: the incrementation count (count of time base). The time elapsed is Accumulated×Time BaseAccumulated\times Time\ BaseAccumulated×Time Base

Outputs Coils:

  • EN: sets to true when timer is engaged.
  • DN: sets to true when timer is done.

TON (timer on delay):

  • Will reset the "Accumulated" value to 0 whenever the input switch/control logic is OFF (in the false state).
  • DN status bit is turned ON when the Accumulated value has reached the Preset value.
  • EN status bit is turned ON when the timer is enabled, and turns off when it is done.

TOF (timer off delay):

  • The exact inverse of TON. Will reset the "Accumulated" value whenever the input control logic is turned ON.
  • DN bit is turned OFF when the timer has expired.
  • EN bit is turned ON when the timer is timing.

RTO (retentive timer on):

  • Identical to TON, but does not reset the "Accumulated" value when the input control logic is switched to OFF.
  • This timer can "pause" counting, and then restart when input control logic is switched back ON.
  • RESET command can be used to reset the Accumulated value.

CTU, CTD Function

Parameters:

  • Counter: the address of the output switch.
  • Preset: the upper counting limit.
  • Accumulated: the count.

Outputs:

  • CU: Count Up.
  • DN: Done.

CTU (Count Up) counts the number of times that input control logic is switched ON up to a Preset value.

  • CU status bit is ON while the counter has not reached the Preset value.
  • When Preset value is reched, the DN bit is switched ON.

CTD (Count Down) decrements the accumulated value each time input control logic is activated.

DIFU, DIFD Function

DIFU is used to turn the desired bit (OneShotOut) on for only one cycle.

DIFD is used to turn the desired bit (OneShotOut) on fo ronly one cycle. If input signal is on, DIFD will generate one cycle pulse on the falling edge on the input signal.

State Diagram

There are 2 types of PLC programs, Non-sequential / combinational logic PLCs are based on truth table. Sequential PLCs are based on state diagrams.

A state diagram contains:

  • State Variable (M): stored in PLC memory, a unique configuration that the system can take.
    • Indegree: inputs
    • Outdegree: outputs
  • Transition and Transition Condition: transitions brings the state machine from one state to another, whena transition condition is true.
  • Entry Action: on entering the state.

Basic Approaches:

  1. Draw circles showing different states of the system.
  2. Connect the states by transition lines, which show the change of states.
  3. Write the identity of boolean variable on transition lines that will cause the change in state.
  4. Extract logic expressions based on this following logic equation: M=X‾(Y+memory)M=\overline{X}(Y+ memory)M=X(Y+memory), where XXX is the turn off signal or RESET bit (i.e. when state bit 1 to 0) and YYY is the turn on signal or SET bit (i.e. state bit 0 to 1).
  5. Implement ladder diagram with the logic equations.

Ladder program from a state diagram:

  1. Number each state uniquely, and allocate enough bits for a state identifier.
  2. State Actions: create a rung, list all outputs and the corresponding states in which they are ON. For each output variable, create a rung ORing.
  3. State Transition: create a rung, using SET and RESET functions to update the state identifier.

You can simplify this with M=X‾(Y+memory)M=\overline{X}(Y+memory)M=X(Y+memory).
