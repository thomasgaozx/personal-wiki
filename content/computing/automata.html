Deterministic Finite Automaton

Regular Language

We say that MMM recognize the language A, L(M)=AL(M)=AL(M)=A, if AAA is the set of all strings that machine MMM accepts, i.e. A={w∣ M accepts w}A=\{w|\ M\ \textrm{accepts}\ w\}A={w∣ M accepts w}.

A machine may accept several strings, but it always recognize only one language.

A language is called a regular language if some finite automaton recognizes it.

Nondeterminism

In deterministic finite automata, each state transition function accept all alphabet symbols. In other words, every state of a DFA always has exactly one exiting transition arrow for each symbol in the alphabet. In a nondeterministic machine, several choices may exist for the next state at any point. When an NFA comes to a state with multiple ways to proceed, after reading that symbol, the machine splits into multiple copies of itself and follows all possibilities in parallel.

  • If the next input symbol doesn't appear on any of the arrows exiting the state occupied by a copy of the machine, the copy of the machine dies.
  • If any one of the copies is in an accept state at some point, the NFA accepts the input string.
  • NFA can have ε\varepsilonε (no input) arrow. If a state with an ε\varepsilonε symbol is encountered, the machine splits a copy immediately following that ε\varepsilonε arrow.

A Nondeterministic finite automaton is a 5-tuple (Q,Σ,δ,q0,F)(Q, \Sigma, \delta, q_0, F)(Q,Σ,δ,q0​,F) where

  1. QQQ: a finite set of states
  2. Σ\SigmaΣ: a finite set of alphabet
  3. δ:Q×Σε⟶P(Q)\delta:Q\times\Sigma_\varepsilon\longrightarrow \mathcal{P}(Q)δ:Q×Σε​⟶P(Q): transition function. Σε=Σ∪{ε}\Sigma_\varepsilon=\Sigma\cup\{\varepsilon\}Σε​=Σ∪{ε}, P(Q)\mathcal{P}(Q)P(Q) denotes the power set of QQQ.
  4. q0∈Qq_0\in Qq0​∈Q: start state
  5. F⊆QF\subseteq QF⊆Q: set of accept states

Let N=(Q,Σ,δ,q0,F)N=(Q, \Sigma, \delta, q_0, F)N=(Q,Σ,δ,q0​,F) be an NFA and w=y1y2⋯ymw=y_1y_2\cdots y_mw=y1​y2​⋯ym​ be a string where yi∈Σεy_i\in\Sigma_\varepsilonyi​∈Σε​. Then NNN accepts www if ∃\exists∃ sequence of states r0,r1,…,rmr_0, r_1, \dots, r_mr0​,r1​,…,rm​ such that

Regular Expressions

We say that RRR is regular expression if RRR is:

  1. aaa for some aaa in the alphabet Σ\SigmaΣ
  2. ε\varepsilonε
  3. ϕ\phiϕ
  4. (R1∪R2)(R_1\cup R_2)(R1​∪R2​), where R1,R2R_1, R_2R1​,R2​ are regular expressions
  5. (R1∘R2)(R_1\circ R_2)(R1​∘R2​), where R1,R2R_1, R_2R1​,R2​ are regular expressions

Equivalence with Finite Automata

Theorem: A language is regular if and only if some regular expression describes it.

Lemma 1: If a language is described by a regular expression, then it is regular.

To prove this lemma, we attempt to convert regular expression to an NFA. If you are unsure where to start, recall the formal definition of regular expression, where there are 6 conditions. Now, all we need to do is to construct 6 state machines:

Generalized Nondeterministic Finite Automaton

Generalized Nondeterministic Finite Automaton (GNFA) is a NFA wherein the transition arrows may have any regular expressions as labels, instead of only memebers of the alphabet or ε\varepsilonε.

Formal Definition: A generalized nondeterministic finite automaton is a 5-tuple (Q,Σ,δ,qstart,qaccept)(Q, \Sigma, \delta, q_{start}, q_{accept})(Q,Σ,δ,qstart​,qaccept​), where

  1. QQQ is the finite set of states
  2. Σ\SigmaΣ is the input alphabet
  3. δ:(Q−{qaccept})×(Q−{qstart})⟶R\delta:(Q-\{q_{accept}\})\times(Q-\{q_{start}\})\longrightarrow\mathcal{R}δ:(Q−{qaccept​})×(Q−{qstart​})⟶R, where R\mathcal{R}R is the collection of all regular expressions over the alphabet Σ\SigmaΣ.
  4. qstartq_{start}qstart​ is the start state
  5. qacceptq_{accept}qaccept​ is the accept state

A GNFA accepts a string www in Σ∗\Sigma^*Σ∗, if w=w1w2⋯wkw=w_1w_2\cdots w_kw=w1​w2​⋯wk​, where wi∈Σ∗w_i\in\Sigma^*wi​∈Σ∗, and a sequence of states q0,q1,…,qkq_0, q_1, \dots, q_kq0​,q1​,…,qk​ exists such that

  1. q0=qstart,qk=qacceptq_0=q_{start}, q_k=q_{accept}q0​=qstart​,qk​=qaccept​
  2. ∀i∈N,wi∈L(Ri)\forall i\in \mathbb{N}, w_i\in L(R_i)∀i∈N,wi​∈L(Ri​), where Ri=δ(qi−1,qi)R_i=\delta(q_{i-1}, q_i)Ri​=δ(qi−1​,qi​)

GNFA Conventions:

  1. The start state has transition arrows going to every other state but no arrows coming in from any other state.
  2. There is only one single accept state, and it has arrows coming in from every other state, but no outgoing arrows.
  3. Except for the start and accept states, one arrow goes from every state to every other state and also each state to itself.

Now to prove that "if a language is regular, then it is descibed by a regular expression."

Our primary objective is to convert a DFA into GNFA by adding a new start state with ε\varepsilonε arrow to the old start state, and by adding a new accept state with ε\varepsilonε arrow from the old accept states. We also add arrows labeled ϕ\phiϕ between states that had no arrows, this won't change anything (just to conform the convention).

Our secondary objective is to shrink GNFA, by "ripping" out one state q∈(Q−{qaccept,qstart})q\in (Q-\{q_{accept}, q_{start}\})q∈(Q−{qaccept​,qstart​}) and repair the machine such that the same language is recognized. This operation is performed iteratively until the new start and accept state are the only 2 states left, with a single arrow connecting the two. The label of this arrow is the droid regex we are looking for.

Let MMM be the DFA for language AAA. We convert MMM to a GNFA GGG using prescribed method. We then define recursive procedure CONVERT(G)(G)(G), which takes a GNFA and returns an equivalent regex:

  3. If k>2k>2k>2, we select any state qrip∈(Q−{qaccept,qstart})q_{rip}\in (Q-\{q_{accept}, q_{start}\})qrip​∈(Q−{qaccept​,qstart​}), and let G′=(Q′,Σ,δ′,qstart,qaccept)G'=(Q',\Sigma,\delta',q_{start}, q_{accept})G′=(Q′,Σ,δ′,qstart​,qaccept​), where Q′=Q−{qrip}Q'=Q-\{q_{rip}\}Q′=Q−{qrip​}.

Non-Regular Language

Pumping Lemma: If AAA is a regular language, then ∃ p∈N\exists\ p\in \mathbb{N}∃ p∈N (pumping length) such that ∀s∈A∧∣s∣≥p,s=xyz\forall s\in A \wedge |s|\ge p, s=xyz∀s∈A∧∣s∣≥p,s=xyz satisfying the following conditions:

  1. xyiz∈A ∀ i≥0xy^iz\in A\ \forall\ i \ge 0xyiz∈A ∀ i≥0
  2. ∣y∣>0|y|>0∣y∣>0
  3. ∣xy∣≤p|xy|\le p∣xy∣≤p

This lemma is actually pretty intuitive. To prove it, let M=(Q,Σ,δ,q1,F)M=(Q,\Sigma,\delta, q_1, F)M=(Q,Σ,δ,q1​,F) be a DFA that recognizes AAA. Let ppp be the number of states of MMM.

  • If no string in AAA are of length at least PPP, then the theorem becomes vacuously true. The 3 conditions hold for all strings of length at least ppp if there aren't any such strings. This is very similar to how implication works.
  • If sss in A has length at least ppp, then it must go through at least p+1p+1p+1 states. This means at least 2 states are repeated, which forms a loop. The string input between the two repeated states is the droid yyy we are looking for.

To use the pumping lemma to prove that a language BBB is not regular, first assume that BBB is regular in order to obtain a contradiction. Use the pumping lemma to guarantee the existence of a pumping length ppp. Then, find a string sss in BBB that has length ppp or greater that cannot satisfy all 3 conditions.

The rules are used to substitute the variables in the string corresponding to the starting variable. Let u,v,wu,v,wu,v,w be strings of variables and terminals, and A→wA\rightarrow wA→w be a rule.

Ambiguity

A derivation of a string www in a grammar GGG is a leftmost derivation if at every step the left omst remaining variable is the one replaced.

A string www is derived ambiguously in context-free grammar GGG if it has two or more different leftmost derivations. Grammar GGG is ambiguous if it generates some string ambiguously.

An inherently ambiguous language is one that can only be generated by ambiguous grammars. Note that some languages can be generated by both ambiguous and unambiguous grammars.
