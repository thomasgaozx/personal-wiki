Turing Machine

Unlike a finite automaton, the turing machine has an infinite tape for read/write. The special states for rejecting and accepting take effect immediately.

A setting of current state, current tape contents, and current head location is called a configuration of the turing machine. For state qqq and two strings u,v∈Γ∗u,v\in\Gamma^*u,v∈Γ∗, we write u q vu\, \bold{q}\, vuqv for the configuration where current state is qqq, the current tape contents is uvuvuv, and current head is the first symbol of vvv.

Language Recognition

A Turing machine MMM accepts input www if ∃ C1,C2,…,Ck:\exists\ C_1,C_2,\dots,C_k:∃ C1​,C2​,…,Ck​:

  1. C1C_1C1​ is the start configuration of MMM on input www
  2. Each CiC_iCi​ yields Ci+1C_{i+1}Ci+1​
  3. CkC_kCk​ is an accepting configuration

The collection of strings that MMM accepts is the language of M\bold{M}M, or the language recognized by M\bold{M}M, denoted L(M)L(M)L(M).

A language is Turing-recognizable aka recursively enumerable language if some Turing machine recognizes it.

When a Turing machine is fed an input, three outcomes are possible: accept, reject, or loop. Loop means the machine does not halt. A turing machine that does not halt is called a decider because they always make a decision to accept or reject. A decider that recognizes some language is said to decide that language.

A language is called a Turing-decidable, aka recursive language, if some Turing machine decides it. Obviously, every decidable language is Turing-recognizable.
