Introduction

A graph GGG consists of a pair of sets V(G),E(G)V(G), E(G)V(G),E(G) where V(G)V(G)V(G) is a finite nonempty set of objects called vertices, and E(G)E(G)E(G) is a set of unordered pair of vertices called edges.

  • Adjacency: two vertices u,vu,vu,v are adjacent if u,v{u,v}u,v is an edge.
  • Incidency: an edge e={u,v}e=\{u,v\}e={u,v} is incident with its two ends uuu and vvv. We also say eee joins uuu and vvv.
  • Neighbours of vertex vvv are vertices adjacent to vvv.
  • Neighbourhood or neighbour set NG(v)N_G(v)NG​(v): the set of all neighbours of vvv
  • Degree of vertex vvv, deg⁡(v)\deg(v)deg(v): the number of edges incident with vvv. Alternatively, the degree of vvv is the size of its neighbourhood, ∣NG(v)∣|N_G(v)|∣NG​(v)∣

For the purpose of this course:

  • We use e=uve=uve=uv to represent the edge e={u,v}e=\{u,v\}e={u,v}. We still think of an edge as a subset of size 222.
  • Edges are unordered, so uv=vuuv=vuuv=vu.
  • We only consider "simple graphs", i.e. no loops (node with an edge to itself) or multiple edges (two nodes have multiple edges in between). Graphs that allow loops and multiple edges are called multigraphs.
  • Graphs we consider are finite.

Isomorphism

Two graphs G1,G2G_1, G_2G1​,G2​ are isomorphic if there is a bijection f:V(G1)→V(G2)f:V(G_1)\rightarrow V(G_2)f:V(G1​)→V(G2​) such that uv∈E(G1)uv\in E(G_1)uv∈E(G1​) if and only if f(u)f(v)∈E(G2)f(u)f(v)\in E(G_2)f(u)f(v)∈E(G2​). Such mapping is called an isomorphism.

Procedure to prove isomorphism doesn't exist:

  1. Find some adjacency properties that holds in one graph but not in the other.
  2. E.g. ∃v∈G1:∀u∈G2,deg⁡(v)≠deg⁡(u)\exists v\in G_1 : \forall u\in G_2, \deg(v)\neq\deg(u)∃v∈G1​:∀u∈G2​,deg(v)​=deg(u)
  3. E.g. G1G_1G1​ contains 3 mutually adjacent vertices, which is not found in G2G_2G2​.
  4. Note that it is insufficient to say that one arrangement is not an isomorphism

Isomorphic relation is an equivalence relation:

  • Reflexive: a graph is isomorphic to itself
  • Symmetric: if G1G_1G1​ is isomorphic to G2G_2G2​, then G2G_2G2​ is isomorphic to G1G_1G1​
  • Transitive: if G1G_1G1​ is isomorphic to G2G_2G2​ and G2G_2G2​ is isomorphic to G3G_3G3​, then G1G_1G1​ is isomorphic to G3G_3G3​.

All graph that are isomorphic to each other form an isomorphism class. We can treat isomorphic graphs as equals, and only one of them represents the whole class.

Simple algorithm to find isomorphism of finite graphs:

  1. Given two lists of vertices, L1,L2L_1, L_2L1​,L2​, for G1G_1G1​ and G2G_2G2​ respectively.
  2. Fixate the order of vertices in L1L_1L1​, and permute L2L_2L2​.
  3. Check iiith vertex from L1L_1L1​ against iiith vertex from L2L_2L2​. If there is any mismatch in adjacency list, go to next permutation iteration.

No polynomial time algorithm has been found so far, but there is also no proof that it is NP-complete (the hardest decision problems)

Connectedness, Components and Cuts

  • Connectedness: a graph GGG is connected if there is a u,vu,vu,v-path for any pair of vertices u,vu,vu,v.
  • Components: a component CCC of graph GGG is a connected subgraph of GGG such that it is a proper subgraph of another connected subgraph of GGG. In other words, a component is a maximally connected subgraph.
    • A connected graph has exactly 1 component, and a graph with only 1 component is connected. A disconnected graph has at least 2 components.
    • There are no edges joining a vertex of a component with a vertex outside the component.
  • Cuts: let X⊆V(G)X\subseteq V(G)X⊆V(G), the cut induced by XXX in GGG is the set of all edges in GGG with exactly one end in XXX.
    • A set of vertices induces a cut. A cut itself is a set of edges.
    • If a graph is disconnected, the cut induced by the vertices of a component is empty.

Theorem (4.8.2): let GGG be a graph and let u∈V(G)u\in V(G)u∈V(G). If a u,vu,vu,v-path exists for each v∈V(G)v\in V(G)v∈V(G), then GGG is connected.

Consequence: we only need to check ∣V(G)∣−1|V(G)|-1∣V(G)∣−1 paths to show connectedness.

  1. Pick a vertex v0v_0v0​ with a special property.
  2. Suppose xxx is an arbitrary vertex with a different property.
  3. Construct a path v0,v1,…,vkv_0, v_1, \dots, v_kv0​,v1​,…,vk​ of length kkk to "reach" xxx

  • Find a pair of vertices where no path between them exists, or
  • Theorem 4.8.5: suppose we have graph GGG,
    1. Select a nonempty proper subset X⊂V(G)X\subset V(G)X⊂V(G)
    2. Show that XXX induces an empty cut. Common technique includes contradiction, or simply demonstrating any edge with one end in XXX has the other end also inXXX, thus there is no edge in the cut induced by XXX.

Eulerian Circuit

An Eulerian circuit, also known as Euler tour in a graph is a closed walk that uses ever edge of the graph exactly once.

A graph that has an Eulerian circuit is not necessarily connected. We may have isolated vertices.

Theorem (4.9.2): let GGG be a connected graph, then GGG has an Eulerian circuit if and only if every vertex in GGG has even degree.

Proof outline:

  • (⇒)(\Rightarrow)(⇒): a closed walk contributes 2 the degree of a vertex for each visit, one for going in, one for going out.
  • (⇐)(\Leftarrow)(⇐): strong induction on the number of edges mmm
    • Base case: m=0m=0m=0, GGG has 0 edges and has a trivial Eularian circuit
    • Induction Hypothesis: if every vertex in GGG has even degree, and GGG has <m< m<m edges, then GGG has an Eulerian circuit.
    • Induction Step: let GGG be a connected graph with mmm edges and all even degrees. Each vertex has ≥2\ge 2≥2 degrees, so by an earlier theorem, there exists a cycle CCC. If CCC contains all edges of GGG, then we are done. Otherwise remove edges of CCC from GGG to get G′G'G′. Both G,CG,CG,C have even-degree vertices, so G′G'G′ does, too. Each component of G′G'G′ has fewer edges than GGG. By induction, every component of G′G'G′ has an Eulerian circuit. Since GGG is connected, each component of G′G'G′ has at least one vertex in common with CCC. Obtain an Eulerian circuit of GGG by attaching the Eulerian circuit of each component to CCC at their common vertex.

Travelling Salesman Problem

Travelling Salesman Problem (TSP): given a graph GGG with edge weights w:E(G)→Rw:E(G)\rightarrow \mathbb{R}w:E(G)→R, find a Hamilton cycle that minimizes total edge weights.

We consider the case that the graph is complete, vertices are on the cartesian plane, and edge weight is the Euclidean distance, and we develop an "approximation" algorithm:

  1. Find an MST, TTT
  2. Double the edges of TTT
  3. Find an Eulerian circuit (from the doubled edges of TTT)
  4. Go along the Eulerian circuit, but skipping repeated vertices.

(From professor Martin Pei's course note)

Claim: let CCC be the Hamilton cycle from the algorithm, let C∗C^*C∗ be the best Hamilton cycle. We claim that w(C)≤2w(C∗)w(C)\le 2 w(C^*)w(C)≤2w(C∗).

Proof: Since (C∗−e)(C^*-e)(C∗−e) is a spanning tree, where eee is an arbitrary edge in the cycle. Firstly, let TTT be the MST, we know w(T)≤w(C∗)w(T)\le w(C^*)w(T)≤w(C∗). Secondly, the weight of the Eulerian circuit is 2w(T)≤2w(C∗)2 w(T)\le 2w(C^*)2w(T)≤2w(C∗). Thirdly, by triangular inequality, we know w(C)≤2w(T)w(C)\le 2w(T)w(C)≤2w(T). Therefore w(C)≤2w(C∗)w(C)\le 2w(C^*)w(C)≤2w(C∗).

Planar Graphs

A Planar Embedding of graph GGG is a drawing of the graph on the plane such that vertices are at different points, and edges intersect only at common endpoints, i.e. edges do not cross. A graph that has a planar embedding is a planar graph.

  • Faces of a planar embedding is a connected region on the plan not separated by edges.
  • Boundary of a face is a subgraph of all vertices and edges that touch the face. Two faces are adjacent if they share at least one edge in their boundaries.
  • Boundary walk of a face in a planar embedding is a closed walk once around hte perimeter of the face boundary.

Euler's Formula

Euler's formula (7.2.1): let GGG be a connected planar graph with nnn vertices and mmm edges. Consider a planar embedding of GGG with sss faces. Then n−m+s=2n-m+s=2n−m+s=2.

Proof: we fix the number of vertices nnn, and prove by induction on the number of edges mmm.

  • Base Case: m=n−1m=n-1m=n−1. A connected graph with n−1n-1n−1 edges is a tree (Corollary 5.2.2). Any planar embedding of a tree has one face s=1s=1s=1. So n−m+s=n−(n−1)+1=2n-m+s=n-(n-1)+1=2n−m+s=n−(n−1)+1=2
  • Induction Hypothesis: assume Euler's formula holds for any connected planar graph with nnn vertices and m−1m-1m−1 edges.
  • Induction Step: consider a connected planar graph with nnn vertices, mmm edges, and sss faces. Let eee be an edge that is not a bridge, then G−eG-eG−e is connected by definition. Also G−eG-eG−e is planar since we can remove eee from a planar embedding of GGG to get a planar embedding of G−eG-eG−e. And G−eG-eG−e has m−1m-1m−1 edges. So Euler's formula holds for G−eG-eG−e by induction. Froma a lemma in the last lecture, since eee is not a bridge, the two sides of eee are different faces. When we remove eee to get an embedding of G−eG-eG−e, these two faces merge into one as they are no longer separated by eee. So this embedding of G−eG-eG−e has s−1s-1s−1 faces. By Euler's formula,

    n−(m−1)+(s−1)=2n-(m-1)+(s-1)=2 n−(m−1)+(s−1)=2

    Then n−m+s=2n-m+s=2n−m+s=2, so Euler's formula holds for GGG.

Exercise: if GGG has ccc components, then Euler's formula becomes n−m+s=1+cn-m+s=1+cn−m+s=1+c.

Matchings

A matching of a graph is a set of edges where no two edges share a common vertex.

  • Saturation: given a matching MMM, a vertex vvv is saturated if vvv is incident with an edge in MMM.
  • Perfect matching: if the matching saturates every vertex.
  • Cover: a cover of a graph GGG is a set of vertices CCC such that each edge in GGG has at least one endpoint in CCC.

Lemma (8.2.1): if MMM is any matching of GGG and CCC in any cover of GGG, then ∣M∣≤∣C∣|M|\le |C|∣M∣≤∣C∣.

Proof: for each edge uv∈Muv\in Muv∈M, at least one of u,vu,vu,v is in CCC (since CCC is a cover). Since no two edges in MMM share a vertex, CCC must include at least ∣M∣|M|∣M∣ distinct vertices, one from each edge of MMM. Hence ∣M∣≤∣C∣|M|\le |C|∣M∣≤∣C∣.

Proof: by Lemma 8.2.1.

Bipartite Matching Algorithm

König's Theorem: in a bipartite graph, the size of a maximum matching is equal to the size of a minimum cover.

We start with a set of definitions and lemmas before tackling König's Theorem:

  • Alternating Path: an alternating path PPP with respect to a matching MMM is a path where consecutive edges alternate being in MMM and not in MMM.
  • Augmenting Path: an alternating path that starts and ends in distinct unsaturated vertices. An augmenting path has odd lengths.

Note that:

  1. Given a matching MMM and an augmenting path with edge set E(P)E(P)E(P), the new matching M′M'M′ is

    M′=(M∪E(P))\(M∩E(P))M'=(M\cup E(P))\backslash (M\cap E(P)) M′=(M∪E(P))\(M∩E(P))

  2. In a bipartite graph, every augmenting path starts and ends with different parts of the bipartition.

We use Bipartite Matching Algorithm to find a matching and cover of the same size for any bipartite graph, which proves König's Theorem.

Algorithm: given a bipartite graph GGG with bipartition (A,B)(A,B)(A,B), and a matching MMM of GGG.

  1. Let X0X_0X0​ be the set of all unsaturated vertices in AAA, let X=X0,Y=ϕX=X_0, Y=\phiX=X0​,Y=ϕ
  2. Find all neighbours of XXX in BBB currently not in YYY
    • If one such vertex is unsaturated, then we have found an augmenting path. We can obtain a larger matching by swapping edges in the augmenting path, go to step 1.
    • If all such vertices are saturated, then put all of them in YYY, add their matching neighbours to XXX, go to step 2.
    • If no such vertices exist , then stop. Our matching is maximum, with minimum cover Y∪(A\X)Y\cup (A\backslash X)Y∪(A\X).

At the end of the algorithm:

  • X0X_0X0​ is the set of all unsaturated vertices in AAA
  • XXX is the set of all vertices in AAA reachable via an alternating path starting with a vertex in X0X_0X0​
  • YYY is the set of all vertices in BBB reachable via an alternating path starting with a vertex in X0X_0X0​

Proof of König's Theorem: let MMM be the matching obtained at the end of the bipartite matching algorithm. let X0,X,YX_0, X, YX0​,X,Y be the result sets at the end of the algorithm. We make a series of claims:

  • Claim 1: there is no edge between XXX and B\YB\backslash YB\Y. Otherwise we can use this edge to extend an alternating path. So every edge has at least one end in A\XA\backslash XA\X or YYY. Hence Y∪(A\X)Y\cup (A\backslash X)Y∪(A\X) is a cover.
  • Claim 2: every vertex in YYY is saturated. Otherwise, we can find an augmenting path, and the algorithm has not terminated. For a matching that saturates a vertex in YYY, the other end is in XXX. Algorithm would add its matching neighbour.
  • Claim 3: every vertex in A\XA\backslash XA\X is saturated. All unsaturated vertices of AAA are in X0X_0X0​, which is in XXX. For a matching that saturates a vertex in A\XA\backslash XA\X, the other end is in B\YB\backslash YB\Y.
  • So the matching edges saturating YYY and A\XA\backslash XA\X are distinct. So ∣M∣=∣Y∣+∣A\X∣|M|=|Y|+|A\backslash X|∣M∣=∣Y∣+∣A\X∣, which is equal to the size of our cover Y∪(A\X)Y\cup (A\backslash X)Y∪(A\X). By Lemma 8.2.2, we have a maximum matching and a minimum cover.
